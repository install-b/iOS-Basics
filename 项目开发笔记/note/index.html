develop(项目开发)

  • 此书记录项目或第三方库等的开发步骤

  • 开发中的各种文档：错误信息文档（status（error:code:105;msc:"请输入合法的邮箱"）;data）、接口文档、需求文档

开发方式：

  • 一般推荐代码+storyboard+xib联合开发，storyboard尽量少用，所用xib图形化开发；
  • storyboard不适合大型项目开发，多人开发容易产生冲突；storyboard的不适用于继承，复用性差

环境部署：

  1. target：如是否支持横竖屏；支持最老的模拟器版本设置（影响系统方法的是否可用）；打开程序时选择隐藏状态栏和全屏

  2. Main Interface（程序启动主窗口加载方式）：在Main Interface中取消main，开发中都是在AppDelegate设置主窗口及加载根控制器。

  3. appicon（图标）：将图片拖动到资源包appicon中，会自动进行配置，根据IOS版本自动归类到对应位置；soptlight是指用于搜索显示的图片；旧项目，严格命名为Icon.png、Icon@2x.png。放到supportingFiles中，不用格外配置；

  4. Launch（启动页面）：

    1. 将需要展示的图片放到assets中，在LaunchScreen.storyboard中设置图片，设置约束（推荐）

    2. 或者在target -- General -- App Icons and Launch Images -- Lauch Images Source -- Use Asset Catalog -- 弹框点击Migrate(不同版本不同，不选Cancel即可)，将需要展示的图片拖到assets新建的文件夹中（不推荐）

    3. 对比：设置LaunchScreen.storyboard比Use Asset Catalog优先级高，推荐通过LaunchScreen.storyboard设置启动图片

      • 因为模拟器和真机的图片是由启动图片确定的，如果通过第二种，会导致显示图片与真实不符。而第一种方法会自动适配机器
      • 通过LaunchScreen加载，可以只选择最大的图片，这样自动布局后可以兼容之前的版本。当然有条件的可以根据不同版本设置不同的启动图片。第二种方法加载必须选择合适的图片！

    4. 旧项目中是对启动图片有规定的命名，放在supportFiles中自动加载

文件布局

  1. 删除原有的vc文件与mainstoryboard

  2. 新文件统一存放在按Classes文件夹中，Classes放在项目文件夹下（项目名必须是英文，否则不能通过审核！）

    1. Classes文件夹：存在各个类的文件夹，类文件夹一般按MVC归档，但一些例外，如Other文件夹，根据情况需要设置

    2. Main(主要)文件夹：存放自定义tabBarVC、自定义导航控制器文件等框架级文件夹，MVC归档

    3. 主界面文件夹：存放tabBarVC下的跳转主界面，MVC归档

    4. 主界面下的各子功能类文件夹，MVC归档

    5. NewFeature(新特性界面):存放新版本的功能展示界面，MVC归档

    6. Login(登录)文件夹：存放登录文件夹，是单例。或者放在other文件夹中，因为不属于任意一个文件夹。

    7. Tool(工具)文件夹：存放Guide（引导）、Save（保存）文件夹、Network(网络请求)等工具性文件夹。

    8. AD（广告）文件夹：存放广告的文件夹

    9. other文件夹：

      • main.m文件
      • Appdelete.h.m文件
      • pch文件
      • Caategory文件夹——存放分类文件
      • Lib文件夹——存放第三方框架文件；

计算总行数\总页数的万能公式

总数 : 2476
每页显示的最大数量 : 35
总页数 :  (2476 + 35 - 1) / 35
pagesCount = (总数  +  每页显示的最大数量 - 1) / 每页显示的最大数量

总数 : 1660
每一行显示的最大数量 : 30
总行数 : (1660 + 30 - 1) / 30
rowsCount = (总数  +  每行显示的最大数量 - 1) / 每行显示的最大数量
