UIKit-Controller

UIApplication

  1. 父类是UIResponder。UIResponder继承自NSObject
  2. 概念：每一个应用程序本身就是UIApplication，是单例
  3. 创建时间：一个iOS程序启动后创建的第一个对象就是UIApplication对象（即应用程序本身）
  4. 作用：UIApplication对象是应用程序的象征，利用UIApplication对象，能进行一些应用级别的操作

UIApplication处理的系统事件

  1. 应用程序图片的提醒数字
    • 设置应用图标右上角的数字,图标需要手动清除,应用程序关闭,不会自动清除.
      app.applicationIconBadgeNumber = 10;
  2. 联网状态
    • 显示联网状态,告诉用户此应用正在联网
      app.networkActivityIndicatorVisible = YES;

  3. 设置状态栏

    • UIApplication管理状态栏.IOS7及之后默认交给控制器,若要交给UIApplication管理，需在info.plist中配置，详见UIViewController管理状态栏

  4. 注册用户通知

     // 创建通知
UIUserNotificationSettings *settings = [UIUserNotificationSettings settingsForTypes:UIUserNotificationTypeBadge categories:nil];
 // 注册用户通知
[app registerUserNotificationSettings:settings];

  5. 打开资源,电话,网页,发短信

    • 打开一个网页资源,UIApplication打开资源的好处:不用判断用什么软件打开,系统会自动根据协议头判断。

        1. 网络资源URL的组成==协议头://主机域名/路径 http://www.baidu.com/abc/1.png
  2. 本地资源URL的组成==协议头:///路径 本机域名可以不写 file:///User/apple/Desktop/1.png

        //十分强大的openURL:方法
  [app openURL:[NSURL URLWithString:@"http://ios.itcast.cn"]];

  //打电话
  UIApplication *app = [UIApplication sharedApplication];
  [app openURL:[NSURL URLWithString:@"tel://10086"]];

  //发短信
  [app openURL:[NSURL URLWithString:@"sms://10086"]];

  //发邮件
  [app openURL:[NSURL URLWithString:@"mailto://12345@qq.com"]];

程序启动流程

一. 加载程序中所有类对象

二. 找到程序入口,执行main函数，然后执行main函数中的UIApplicationMain函数

   main -> UIApplicationMain

三. UIApplicationMain底层处理：

1. 创建UIApplication对象或其子类对象
2. 创建UIApplication的代理对象或其子类对象,而且给UIApplication对象的代理属性赋值
3. 开启主运行循环,接收事件并让程序一直运行
4. 加载info.plist,判断有木有指定main.storyboard,如果指定就会去加载storyboard
5. 程序启动完成后创建窗口UIWindow。窗口被AppDelegate用强指针指向，所以不会销毁。
6. 加载main.storyboard并实例化控制器（在创建窗口之后，详见苹果官方文档）
7. 让控制器成为窗口的根控制器，并让窗口成为主窗口并显示,在窗口显示前，会懒加载根控制器的view，最后展示界面
8. 界面显示

UIApplicationMain函数

  • main函数中执行了UIApplicationMain这个函数
int main(int argc, char * argv[]) {
    @autoreleasepool {
        /**
         *  应用程序主函数，死循环
         *
         *  @param argc   直接传递给UIApplicationMain进行相关处理即可
         *  @param argv   直接传递给UIApplicationMain进行相关处理即可
         *  @param nil    principalClassName：指定应用程序类名（app的象征），该类必须是UIApplication(或子类)。
            如果为nil,则用UIApplication类作为默认值
         *  @param [AppDelegate class] delegateClassName：指定应用程序的代理类，该类必须遵守UIApplicationDelegate协议,并让其成为程序类的代理;
            delegateClassName不能传nil,这里传nil,意味着application没有代理,就无法监听系统的事件,系统的事件都没法监听,就无法创建并让窗口显示
         *
         *  @return 程序正常退出时UIApplicationMain函数才返回
         */
        return UIApplicationMain(argc, argv, nil, NSStringFromClass([AppDelegate class]));
    }
}
