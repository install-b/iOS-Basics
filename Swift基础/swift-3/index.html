Swift中数据类型

  • Swift中的数据类型也有:整型/浮点型/对象类型/结构体类型等等
  • 数据类型关键字首字母都是大写
    • 整型
      • 有符号
          Int8 : 有符号8位整型
  Int16 : 有符号16位整型
  Int32 : 有符号32位整型
  Int64 : 有符号64位整型
  Int : 和平台相关(默认,相当于OC的NSInteger)
      • 无符号
          UInt8 : 无符号8位整型
  UInt16 : 无符号16位整型
  UInt32 : 无符号32位整型
  UInt64 : 无符号64位整型
  UInt : 和平台相关(常用,相当于OC的NSUInteger)(默认)
    • 浮点型
      • Float : 32位浮点型
      • Double : 64浮点型(默认)
// 定义一个Int类型的变量m,并且赋值为10
var m : Int = 10
// 定义一个Double类型的常量n,并且赋值为3.14
let n : Double = 3.14

Swift中的类型推导

  • Swift中任何一个标识符都有明确的类型
  • 注意:
    • 如果定义一个标识符时直接赋值,那么其类型可以省略.
    • 因为Swift有类型推导,编译时会自动根据赋值类型来决定前面标识符的数据类型(Swift独有)
    • 可以通过 option + 鼠标左键 来查看变量的数据类型
// 定义变量时没有指定明确的类型
// 但因为赋值i=20 ===>由20为整型->推导i为整型
var i = 20
// 错误写法:如果之后赋值给i一个浮点型数值,则会报错
// i = 30.5

// 正确写法
var j = 3.33
j = 6.66

// 错误写法：定义时没有指定表达式的类型, 也没有赋予初始值
// 编译器会报错(type annotation missing in pattern)

Swift类型转换

  • Swift中在进行基本运算时必须保证类型一致,否则会出错
    • 相同类型之间才可以进行运算
    • 因为Swift中没有隐式转换
  • 数据类型的转化
    • Int类型转成Double类型:Double(标识符)
    • Double类型转成Int类型:Int(标识符)
    • Double和CGFloat也需要转换
let a = 10
let b = 3.14

// 错误写法-不存在隐���式转换
// let c = a + b
// let c = a * b

// 正确写法
let c = Double(a) + b
let d = a + Int(b)

// 结合数字类常量和变量不同于结合数字类字面量。
// 字面量之间运算可以是不同类型
// 字面量8可以直接和字面量3.141相加，因为数字字面量本身没有明确的类型。它们的类型只在编译器需要求值的时候被推测
let sum2 = 10 + 10.1

// CGFloat/Double
// Double和CGFloat也需要转换
let point = CGPoint(x: 10.10, y: 20.20)
let temp = point.x
let sum3 = Double(temp) + b
