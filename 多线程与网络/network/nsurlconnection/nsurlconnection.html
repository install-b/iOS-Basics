## NSURLConnection（过期不用，了解）

  • 父类是NSObject
  • 发送get、post请求的区别是可变请求对象的设置，不设置默认是get请求。
  • 可以开启任务，取消任务，但不能暂停任务

#### NSURLConnection网络请求常用类（过期不用，了解） 01 NSURL：请求地址

02 NSURLRequest

03 NSMutableURLRequest：NSURLRequest的子类

04 NSURLConnection 4.1 负责发送请求，建立客户端和服务器的连接 4.2 发送数据给服务器，并收集来自服务器的响应数据

#### NSURLConnection发送网络请求（过期不用，了解） - 详见“Network（发送网络请求）-NSURLConnection（过期）-NSURLConnection--block（过期）” - 详见“Network（发送网络请求）-NSURLConnection--delegate（过期）”

#### NSURLConnection代理与线程、Runloop

（1）NSURLConnection代理与Runloop：

    01 下述两种方法设置代理，会自动的发送请求，系统内部会把connect添加到当前线程的RunLoop中，在默认模式下执行
    // [[NSURLConnection alloc]initWithRequest:request delegate:self];
    // [NSURLConnection connectionWithRequest:request delegate:self];

    02 下述方法设置代理，startImmediately为NO的时候，该方法不会自动发送请求
    NSURLConnection *connect = [[NSURLConnection alloc]initWithRequest:request delegate:self startImmediately:NO];
    //手动通过代码的方式来发送请求
    //注意该方法内部会自动的把connect添加到当前线程的RunLoop中，在默认模式下执行
    [connect start];

（2）NSURLConnection代理与线程：

    //说明：默认情况下，代理方法会在主线程中进行调用（为了方便开发者拿到数据后处理一些刷新UI的操作不需要考虑到线程间通信）

    //设置代理方法的执行队列
    [connect setDelegateQueue:[[NSOperationQueue alloc]init]];

（3）开子线程发送网络请求的注意点，适用于自动发送网络请求模式


//在子线程中发送网络请求-调用start方法发送
-(void)createNewThreadSendConnect1
{
    //1.创建一个非主队列
    NSOperationQueue *queue = [[NSOperationQueue alloc]init];

    //2.封装操作，并把任务添加到队列中执行
    [queue addOperationWithBlock:^{

        //2-1.确定请求路径
        NSURL *url = [NSURL URLWithString:@"http://120.25.226.186:32812/login?username=dd&pwd=ww&type=JSON"];

        //2-2.创建请求对象
        NSURLRequest *request = [NSURLRequest requestWithURL:url];

        //2-3.使用NSURLConnection设置代理，发送网络请求
        NSURLConnection *connection = [[NSURLConnection alloc]initWithRequest:request delegate:self startImmediately:YES];

        //2-4.设置代理方法在哪个队列中执行，如果是非主队列，那么代理方法将在子线程中执行
        [connection setDelegateQueue:[[NSOperationQueue alloc]init]];

        //2-5.发送网络请求
        //注意：start方法内部会把当前的connect对象作为一个source添加到当前线程对应的runloop中
        //区别在于，如果调用start方法开发送网络请求，那么再添加source的过程中，如果当前runloop不存在
        //那么该方法内部会自动创建一个当前线程对应的runloop,并启动。这话是错的。必须选择代理方法的执行线程再开启connection，这样才会执行代理方法，代理方法会在新的线程中执行。可以在start后面打印一句话测试，如果开始了runloop，那句话应该最后打印！
        [connection start];

    }];
}

//在子线程中发送网络请求-自动发送网络请求
-(void)createNewThreadSendConnect2
{
    NSLog(@"-----");
    //1.创建一个非主队列
    NSOperationQueue *queue = [[NSOperationQueue alloc]init];

    //2.封装操作，并把任务添加到队列中执行
    [queue addOperationWithBlock:^{

        //2-1.确定请求路径
        NSURL *url = [NSURL URLWithString:@"http://120.25.226.186:32812/login?username=dd&pwd=ww&type=JSON"];

        //2-2.创建请求对象
        NSURLRequest *request = [NSURLRequest requestWithURL:url];

        //2-3.使用NSURLConnection设置代理，发送网络请求
        //注意：该方法内部虽然会把connection添加到runloop,但是如果当前的runloop不存在，那么不会主动创建。
        NSURLConnection *connection = [NSURLConnection connectionWithRequest:request delegate:self];

        //2-4.设置代理方法在哪个队列中执行，如果是非主队列，那么代理方法将再子线程中执行
        [connection setDelegateQueue:[[NSOperationQueue alloc]init]];

        //2-5 创建当前线程对应的runloop,并开启；如果不设置代理方法的执行线程，默认在当前线程中执行
       [[NSRunLoop currentRunLoop]run];
    }];
}

登录小案例

  • 注意细节：
    1. 本地判断：在发送网络请求时，先在本地对账号密码是否符合一定规则（例如是否不为空，密码是否是数字和英文组合灯）判断，提高性能。
    2. 发送POST请求：账号和密码是敏感信息，发送POST请求
    3. 解析网络请求：解析网络请求（直接解析、JSON解析、XML解析）并刷新UI；账号密码有误需截取响应的字符串返回给用户，可通过NSString的 substringWithRange方法
    4. 弹窗提示：本地判断不通过，弹框提示原因；发送网络请求时弹框提示；网络返回数据解析后弹框提示
#import "ViewController.h"
#import "SVProgressHUD.h"

@implementation ViewController

- (IBAction)loginBtnClick:(id)sender {

    //0.拿到用户输入的参数
    NSString *usernameStr = self.usernameTextField.text;
    NSString *pwdStr = self.pwdTextField.text;

    //先设置遮罩的背景类型为灰色
    [SVProgressHUD setDefaultMaskType:SVProgressHUDMaskTypeBlack];

    //判断参数是否有值,如果没有值的话那么提示用户重新输入并返回
    if (usernameStr.length == 0) {
        [SVProgressHUD showErrorWithStatus:self.usernameTextField.placeholder];
        return;
    }
    if (pwdStr.length == 0) {
        [SVProgressHUD showWithStatus:self.pwdTextField.placeholder];
        return;
    }

    //1.确定url
    NSURL *url = [NSURL URLWithString:@"http://120.25.226.186:32812/login"];

    //2.创建可变的请求对象
    NSMutableURLRequest *request = [NSMutableURLRequest requestWithURL:url];

    //3.修改请求方法
    request.HTTPMethod = @"POST";

    //4.设置请求体
    NSString *param = [NSString stringWithFormat:@"username=%@&pwd=%@",usernameStr,pwdStr];
    request.HTTPBody = [param dataUsingEncoding:NSUTF8StringEncoding];

//    [SVProgressHUD show];
//    [SVProgressHUD showWithStatus:<#(NSString *)#> maskType:<#(SVProgressHUDMaskType)#>]

    [SVProgressHUD showWithStatus:@"正在登录"];

    //5.发送请求
    [NSURLConnection sendAsynchronousRequest:request queue:[NSOperationQueue mainQueue] completionHandler:^(NSURLResponse * _Nullable response, NSData * _Nullable data, NSError * _Nullable connectionError) {

        if (connectionError == nil) {
              //6.解析数据
            NSString *result = [[NSString alloc]initWithData:data encoding:NSUTF8StringEncoding];
            NSLog(@"%@",result);
            [NSThread sleepForTimeInterval:1.0];
            /*
             {"error":"用户名不存在"}
             {"error":"密码不正确"}
             {"success":"登录成功"}
            */
            // [SVProgressHUD dismiss];
            //截取字符串
            NSUInteger loc = [result rangeOfString:@"\":\""].location + 3;
            NSUInteger len = [result rangeOfString:@"\"}"].location - loc;
            NSRange range = NSMakeRange(loc, len);
            NSString *msg = [result substringWithRange:range];

            //7.提示用户登录信息
            if ([result containsString:@"error"]) {
                [SVProgressHUD showErrorWithStatus:msg];
            }else
            {
                [SVProgressHUD showSuccessWithStatus:msg];
            }
        }
    }];
}
